幸运破解器需要 Root 权限才能工作！

颜色分类说明：

绿色 - 此应用有很大机会被破解；
黄色 - 此应用在以下目录发现可用自定义破解 /sdcard/LuckyPatcher/；
青色 - 此应用含有谷歌广告；
洋红 - 此应用已添加到启动列表；
紫色 - 此应用含有谷歌内购；
红色 - 此应用几乎不可能被破解。因为应用拥有两部分（应用和专业版密钥），在这种情况下，需要同时破解两部分而不是其中一部分；
橙色 - 系统应用。对系统应用执行任何操作时，请务必谨慎，它可能对应用或操作系统造成严重的不良后果。

启动列表 - 开机后自动应用破解的应用列表。此列表仅在应用需要修改库文件“.so”时才使用。添加到启动列表需要在自定义破解中声明。

四叶草 -此应用已应用幸运破解器；
星标 - 意味着应用的虚拟机缓存已被修复（已创建 odex 更改），相反则意味着应用将保持当前的状态，直至应用被再次更新或删除。如果此 odex 使用“移除 odex 更改”来移除，那应用将会完全恢复到被应用破解前的状态。当您更新应用时，问题就会出现：电子市场将会提示空间不足，此时，运行幸运破解器并关闭，再尝试安装该应用。如果这样的错误出现在已卸载的应用中，请选择“故障排除 -> 移除修复和备份（更新应用）”- 所有修复的应用虚拟机缓存将会被删除！

上下文菜单：
移除授权验证（自动模式） - 尝试移除应用授权验证。将会使用尽量小的破解模块来进行破解以确保应用的最大稳定性。
移除授权验证（自动模式（逆向）） - 此模式是“自动模式”的逆向破解模式。如果“自动模式”破解后没有效果，可以尝试使用此项进行破解。
移除授权验证（极限自动模式） - 尝试完全移除应用授权验证。将会使用所有的破解模块来进行破解，但破解后可能导致应用不稳定，在使用过程中会产生更多的错误。重启应用，变更或禁用屏幕方向也许会有所帮助。
移除授权验证（手动） - 尝试手动移除应用授权验证。在自动模式破解失败后，手动选择破解模式来进行强力移除应用授权验证。使用手动模式时，先备份应用，然后选择第一种“破解”方式，破解完成后选择“启动”以验证破解是否成功。如果失败，则选择“恢复”从下一种“破解”方式继续以上操作；如果成功，则选择“固定破解”并“保存为自定义破解”。自定义破解有可能在之后的版本中失效。
移除授权验证（选择破解模式） - 选择一组破解模块来对应用进行破解。
移除谷歌广告 - 尝试移除应用自带的谷歌广告。有时候应用会处于未发现任何广告的状态，您需要从未找到广告标签的应用中进行移除广告。
禁用广告活动 - 手动禁用应用活动。您可以从活动描述中获取活动目的，如果应用工作不正常请重启活动。
应用内购及授权验证库模拟支持 - 添加模拟应用内购功能，此功能可以在开关中打开。
禁用组件“权限禁用”模式 - 移除选定的权限、服务、活动等。此功能是 Android 内置且安全。对于应用修改后的崩溃，可通过启用权限以修复。
更改权限（危险模式） - 移除应用权限（编辑系统文件 packages.xml）。请慎用，此功能也许会导致循环重启或其他不良后果。操作前请务必备份系统。
更改权限和活动项（安全模式） - 通过重建应用以移除应用权限。此操作不会更改应用签名，仅适用于系统应用或“禁用签名验证”后的全部应用。
更改权限和活动项并重建与安装 - 类似于安全模式，但此操作不会保留原始签名。对于存在签名验证的应用可能无法正常工作。
创建修改的 APK - 基于可用的破解来创建一个 APK 安装包。请注意：重建的 APK 文件和破解的已安装应用效果并不同，由于破解后的应用可以很简易的测试并使用正常，但安装重建的 APK 文件使用效果并非一样。
移除 odex 更改 - 移除 odex 文件及包含其更改，将应用恢复到初始状态。
移除选定保存的内购 - 在谷歌内购模拟中移除保存的内购。
恢复 - 从 /sdcard/LuckyPatcher/Backup/ 文件夹恢复备份的 APK 文件。
备份 - 备份 APK 文件到 /sdcard/LuckyPatcher/Backup/ 文件夹。
重启时应用破解 - 在新版的幸运破解器中所有更改保存到应用的 odex 文件中，此文件重启时不变，故此设置已不再起作用。此项仅在应用需要修改库文件“.so”时才使用，使用的自定义破解需要声明此设置。
手动破解 - 通过输入的模块选择破解文件，对于调试应用很有作用。
应用 odex 化 - 对应用创建未更改的 odex 文件。
移动到 /system/app - 转换用户应用为系统应用。
分享应用 - 对选定应用创建备份的 APK 文件并分享。
复制更改到虚拟机缓存 - 在默认模式下的所有更改将会保留 odex 文件。但某些 odex 文件却不能工作于相应的应用（例如：当移动应用到外部 SD 卡时），因此，复制更改到虚拟机缓存就可以解决此问题。但是，如果您只想重新安装此应用，可以直接恢复它。
备份重新安装的 APK 文件 - 复制 APK 文件到 SD 卡的“LuckyPatcher”数据文件夹。复制成功后可以从这个 APK 文件重新安装（恢复）相应的应用。
删除虚拟机缓存 - 删除此应用在虚拟机缓存创建后的 odex 文件以释放空间。如果 odex 文件不能工作于此应用，请重新安装此应用。

破解说明：
移除授权验证：
- 只能用于开发者签名的原始 APK 文件；
- 长按应用并选择“移除授权验证”；
- 选择“自动模式”并连接网络测试应用；
- 若破解无效，再选择“自动模式（逆向）”或“极限自动模式”；
- 如果您够幸运，应用将会被破解；
- 点击“移除 odex 更改”可恢复应用到初始状态。

移除谷歌广告：
- 长按应用并选择“移除谷歌广告”；
- 选择“破解谷歌广告”；
- 运行并连接网络测试应用；
- 如果广告依旧存在，选择“禁止广告活动”，找到广告活动并禁用；
- 如果您够幸运，谷歌广告将会被移除；
- 点击“移除 odex 更改”或“恢复”可恢复应用到初始状态。

自定义破解：
长按应用并选择“自定义破解”；
运行并连接网络测试应用；
- 如果您够幸运，破解将有效；
- 点击“移除 odex 更改”或“恢复”可恢复应用到初始状态。

更改权限：
- 长按应用并选择“更改权限”；
- 单击不需要的权限以禁用 （红色）；
- 单击需要的权限以启用 （绿色）；
- 最后必须点击“立即重启应用更改”。

创建修改的 APK：
- 长按应用并选择“创建修改的 APK”；
- 选择修改的 APK 破解；
- 已修改的安装文件将会保存到 /sdcard/LuckPatcher/Modified/；
- 请注意：重建的 APK 文件和破解后的应用效果并不同，由于应用破解后可以很简易的测试自身的更改并使用正常，但安装重建的 APK 文件使用效果并非一样。

备注：当破解应用时，幸运破解器会关闭其应用以应用新的更改。但有时您可能需要重新启动设备或使用第三方工具来终止应用的进程以应用破解。
